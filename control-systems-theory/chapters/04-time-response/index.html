Time-Domain Response

Transient & Steady-State Specifications

1. Transient Specifications

Performance in control design is often specified in the time domain using a step input. For a standard underdamped second-order system:

$$ y(t) = 1 - \\frac{e^{-\\zeta\\omega_n t}}{\\sqrt{1-\\zeta^2}} \\sin(\\omega_d t + \\cos^{-1}\\zeta) $$
  • Rise Time ($t_r$): Time to go from 10% to 90%. Approx: $t_r \\approx \\frac{1.8}{\\omega_n}$
  • Peak Time ($t_p$): Time to reach the first maximum. $t_p = \\frac{\\pi}{\\omega_d}$
  • Settling Time ($t_s$): Time to stay within 2% of final value. $t_s \\approx \\frac{4}{\\zeta\\omega_n}$
  • % Overshoot ($M_p$): Maximum peak past final value.
$$ M_p = 100 \\times e^{\\frac{-\\pi\\zeta}{\\sqrt{1-\\zeta^2}}} \\% $$ (4.1)

2. System Type and Steady-State Error

Steady-state error $e_{ss}$ evaluates how well a system tracks an input after transients die out.

Final Value Theorem:

$$ e_{ss} = \\lim_{t \\to \\infty} e(t) = \\lim_{s \\to 0} s E(s) $$ (4.2)

The System Type is defined as the number of pure integrators (poles at origin $s=0$) in the open-loop transfer function $G(s)$.
